## Context

WyrdWerk already operates a fully functional private Agentic Memory Hub -- a Git-backed repository where every AI conversation is saved as structured markdown with YAML frontmatter. Seven coding agents (Cursor, Claude Code, Codex, OpenCode, Pi, Droid, Hermes) write to this private store daily, producing 80+ indexed conversations. The private repo contains real user data, real Tailscale IPs, real API tokens, and personal identifiers. This session created a sanitized, public-facing skeletal framework from that working system -- stripped of all private data, with placeholders for anyone who wants to replicate the architecture.

## Key Discussion Points

1. **Private Hub Precedent**
   - Working private repository with 85+ conversations across 7 agents
   - Agents: Cursor (IDE), Claude Code (CLI), Codex (CLI), OpenCode (CLI), Pi (CLI), Droid (factory), Hermes (Discord/Slack)
   - Daily workflow: boot at session start, save at end, index once per day
   - MCP server running centrally enabling cross-repo operation from any project directory

2. **Sanitization for Public Release**
   - All real IP addresses replaced with `<YOUR_VPS_TAILSCALE_IP>` placeholders
   - All real names replaced with `<YOUR_NAME>` placeholders
   - All real repo URLs replaced with `<YOUR_USERNAME>/<YOUR_PRIVATE_REPO>` patterns
   - Private repo name references (`agentic-memory-hub`) scrubbed from search paths, error messages, and templates
   - Authentication tokens and bearer tokens replaced with placeholder instructions
   - Git remote references sanitized in `.cursorrules`, `SKILL.md`, agent configs, and MCP server code

3. **Framework Skeleton Delivered**
   - **SKILL.md**: Canonical specification for conversation format, detection rules, and auto-commit workflow
   - **AGENTS.md**: Generic guidance applicable to all agents (save, boot, index, digest workflows)
   - **Per-agent configs**: `.cursorrules`, `.claude/CLAUDE.md`, `.codex/CODEX.md`, `.opencode/OPENCODE.md`, `.pi/PI.md`, `.hermes/HERMES.md`, `.droid/DROID.md`
   - **Six distributed skills**: `memory-bridge`, `memory-bridge-boot`, `memory-bridge-index`, `memory-bridge-digest`, `memory-bridge-import`, `llm-wiki-compile`
   - **MCP server**: Full Python implementation (`scripts/memory_bridge/mcp_server.py`) with dual-transport support (SSE on port 8080, Streamable HTTP on port 8081)
   - **Deterministic CLI**: `bin/memory-bridge` wrapping stdlib Python for index rebuild, freshness checks, lexical search, and import
   - **Templates**: `templates/conversation.md` with YAML frontmatter schema
   - **Documentation**: `docs/mcp-setup.md` (per-agent transport matrix), `docs/schema.md` (frontmatter reference), `docs/cli.md`, `docs/import-workflows.md`, `docs/index-format.md`

4. **MCP Server Architecture**
   - Dual-port design: SSE (`/sse`) for legacy clients, Streamable HTTP (`/mcp`) for modern clients
   - 11 exposed tools: `save_conversation`, `update_conversation`, `search_conversations`, `digest_conversations`, `list_conversations`, `show_conversation`, `get_status`, `rebuild_index`, `check_index`, `import_conversations`, `sync_skills`
   - Targeted git staging: `save_conversation` and `update_conversation` stage only the target file + `INDEX.md` (prevents surprise commits of unrelated working-tree changes)
   - `rebuild_index` intentionally uses `git add .` since it may touch multiple index sidecars
   - Bearer token authentication supported (defense in depth even on private networks)
   - `asyncio.Lock` for concurrent write safety with auto-retry via `git pull --rebase`

5. **Skill Design Philosophy**
   - Markdown instructions, not code -- agents follow the skill text to execute workflows
   - Self-contained: each skill works independently; no cross-dependencies
   - Agent-agnostic: same format for all agent types
   - Deterministic: lexical search + topic aliases, no embeddings, no API keys, works offline
   - Human-readable: plain markdown, inspectable, portable

6. **Index System**
   - `INDEX.md` machine-generated at repo root -- full rewrite on every rebuild
   - Scans all conversation frontmatter for metadata
   - Scans all conversation bodies for open item counts
   - Deterministic lexical scoring + `config/topic_aliases.json` for semantic expansion
   - `.index/` JSON sidecars gitignored (regenerable locally)

7. **What Changed After Initial Creation**
   - MCP fallback paths added to three skills (`memory-bridge`, `memory-bridge-boot`, `memory-bridge-index`) so agents can operate remotely without local repo checkout
   - `update_conversation` tool added to close the append-only gap -- agents can now edit existing conversations without local repo access
   - Documentation synced: tool count updated (10 -> 11), git staging behavior documented, Droid skill path fixed (`~/.droid/skills` -> `~/.agents/skills`)
   - Private agent configs updated to list actual MCP tool names instead of stale skill trigger names
