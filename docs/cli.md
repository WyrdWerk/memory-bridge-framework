# `bin/memory-bridge` CLI

Stdlib Python only (`scripts/memory_bridge`). Ensure executable bit: `chmod +x bin/memory-bridge`.

`PYTHONPATH` is set automatically to `<repo>/scripts`.

## Commands

| Command | Purpose |
|---------|--------|
| `status` | Repo path, conversation count |
| `health` | Quick import + cwd sanity |
| `demo` | Short sample outputs |
| `check [--json]` | Index freshness vs disk + hashes |
| `index rebuild [--no-sidecars]` | Rewrite `INDEX.md` and optional `.index/` |
| `index check [--json]` | Alias for freshness |
| `search QUERY... [--limit N]` | Lexical ranking over stems, topics/keywords/repos, checklist text, body |
| `digest TOPIC... [--limit N]` | Same ranker biased toward topic-like fields |
| `show NEEDLE` | Locate a substring in paths / index table |
| `import PATHS... [--agent-id …] [--format …] [--user …] [--dry-run]` | Wrap exports into canonical frontmatter + template body |

Use `--dry-run` on `import` to print the synthesized markdown without writing.

## Topic aliases

`config/topic_aliases.json` expands query terms deterministically (e.g. `{ "legal": ["contract", "llp"] }`).

## Ranking (no embeddings)

Stem overlap scoring only — deterministic and offline. For reproducible rankings use the CLI; interactive agents should mirror the same rules (see `skills/memory-bridge-digest/SKILL.md`).

`search` and `digest` re-parse all conversation markdown files on each invocation. `.index/` sidecars are generated by `index rebuild` but are not read by the ranker — that optimization is deferred.
