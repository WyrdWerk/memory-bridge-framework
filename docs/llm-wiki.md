# LLM Wiki Integration

**Status:** Feature branch (feature/llm-wiki-integration)  
**Scope:** Add Karpathy-style LLM Wiki as compounding knowledge layer alongside existing conversation index

## Overview

This document describes the LLM Wiki integration added to the Agentic Memory Hub. The wiki provides a **synthesis layer** on top of the existing `conversations/` directory and `INDEX.md`.

### Key Principle

> **INDEX.md is NOT replaced.** It remains the machine-generated table of contents for finding specific conversations. The wiki is a **new capability** for compounding knowledge across multiple conversations.

| Component | Purpose | Data Flow |
|-----------|---------|-----------|
| `conversations/` | Raw session summaries (47 files) | Generated by agents via `save-to-memory-bridge` |
| `INDEX.md` | Machine-generated registry | Deterministic index for lookup |
| `wiki/` | **NEW** Compounding knowledge synthesis | Ingest → Compile → Cascade updates |
| `raw/` | **NEW** Immutable source material | Conversations copied when ingested |

## Directory Structure

```
memory-bridge-framework/
├── conversations/YYYY/MM/DD/*.md   # Existing: timestamped session summaries
├── INDEX.md                       # Existing: machine-generated TOC
├── raw/                           # NEW: Immutable source material
│   └── topic/
│       └── YYYY-MM-DD-source-slug.md
├── wiki/                          # NEW: Compiled knowledge articles
│   ├── topic/
│   │   └── concept-name.md
│   ├── index.md                   # Wiki TOC (different purpose from INDEX.md)
│   └── log.md                     # Operation audit trail
└── docs/
    └── llm-wiki.md                # This file
```

## Workflow: Ingest → Compile → Query

### 1. Ingest

User says: "Ingest MCP conversations into wiki"

1. Copy relevant conversations from `conversations/` to `raw/topic/`
2. Preserve metadata (source path, date, agent, topics)
3. Do NOT modify original conversations

### 2. Compile

LLM synthesizes multiple sources into `wiki/topic/concept-name.md`:

- **Decision:** New concept or merge into existing article?
- **Content:** Synthesize key points, decisions, action items
- **Cross-references:** Link to related wiki articles
- **Raw citations:** Link back to `raw/` sources

### 3. Cascade Updates

After compiling primary article:
- Scan related wiki articles for content affected by new source
- Update cross-references and "See Also" sections
- Refresh `wiki/index.md` entries

### 4. Query

User asks: "What do I know about MCP server setup?"

- Read `wiki/index.md` to locate relevant articles
- Read article(s) and synthesize answer
- Cite sources with links: `[Article](wiki/topic/article.md)`

## Comparison: INDEX.md vs Wiki Article

| Aspect | INDEX.md Entry | Wiki Article |
|--------|---------------|--------------|
| **Purpose** | Find the conversation | Understand the concept |
| **Content** | Topics list, excerpt, file path | Synthesized knowledge, patterns, cross-references |
| **Open items** | Count only | Detailed checklist with context |
| **Updates** | Regenerated on index rebuild | Compounds with new ingests |
| **Scope** | One conversation | Multiple conversations + synthesis |

## When to Use Each System

### Use INDEX.md when:
- You need to find a specific conversation from a known date
- You want to see all open items across conversations
- You're checking the freshness of the repository

### Use Wiki when:
- You want to understand a cross-cutting theme (e.g., "MCP architecture")
- You need synthesized knowledge from multiple sessions
- You want patterns, decisions, and action items in one place

## Implementation Notes

### Determinism Preserved

Following the Memory Bridge philosophy:
- ✅ No embeddings required
- ✅ No API keys needed
- ✅ No external services
- ✅ Pure markdown + YAML frontmatter
- ✅ Git-tracked (everything except `.gitkeep` markers)

### File Formats

**Raw source format:**
```yaml
---
source: conversations/2026/05/06/20260506-160859-opencode.md
collected: 2026-05-08
published: 2026-05-06
author: opencode
topics:
  - mcp-server-architecture
---
```

**Wiki article format:**
```yaml
---
title: Memory Bridge MCP Server
created: 2026-05-08
updated: 2026-05-08
topic: mcp
sources: OpenCode; Hermes; Codex
raw: ../../raw/mcp/2026-05-06-...
---
```
